# Setuna UI 重新设计 — 双模式架构

## 设计理念

**截图工具的本质**：Setuna 是截图+贴图工具，截图浮动窗永远是核心。LLM助手、待办管理是辅助功能，不应喧宾夺主。

**双模式设计**：
- **迷你模式**（常驻桌面）：260px 宽度，显示待办数量 + LLM会话状态摘要，不干扰工作
- **全屏工作台**（需要时展开）：960x640，包含聊天/待办/历史/设置四个Tab

## 被合并/删除的窗口

| 旧窗口 | 去向 | 原因 |
|--------|------|------|
| OptionWindow (8个Tab) | 工作台内"设置"Tab，平铺式 | Tab嵌套太深，改为Section平铺 |
| TodoWindow | 工作台内"待办"Tab | 合并为单一入口 |
| CompactTodoWindow | 迷你窗口 | 迷你窗口本身就是"精简版" |
| HistoryWindow | 工作台内"历史"Tab | 合并为单一入口 |
| MainWindow (隐藏宿主) | 移除，App.xaml.cs直接管理 | 无实际窗口，纯宿主无意义 |

## 保留不变的窗口

| 窗口 | 保留原因 |
|------|----------|
| CaptureWindow | 截图核心，独立浮动 |
| ScrapWindow (贴图) | 核心功能 |
| PaintWindow / PenTool / TextTool | 绘图工具，需要独立浮动 |
| StyleEditWindow / StyleItemSelectWindow | 配置型，低频使用 |

## 新增：迷你窗口交互规则

### 交互设计
- **标题栏**：可拖动（DragMove），右上角展开按钮（⛶）
- **待办区域**：checkbox直接切换完成状态 + 快速添加输入框
- **LLM状态**：显示会话数量和最近会话名，点击跳转全屏聊天Tab
- **底部按钮**：快捷跳转到全屏工作台的对应Tab

### 位置
- 默认：屏幕右下角（SystemParameters.WorkArea定位）
- 可拖动到任意位置
- 记住上次位置（SetunaOption中存储）

## 新增：截图右键"发起对话"

在浮动截图的右键菜单中添加"发起对话"：
1. 点击后在工作台中创建新会话
2. 将截图作为图片附件加入会话
3. 自动切到聊天Tab
4. LLM收到截图内容（通过OCR工具或直接base64）

## 技术实现要点

1. **迷你窗口**：`WindowStyle=None`, `AllowsTransparency=True`, `Topmost=True`
2. **全屏工作台**：普通窗口，`ResizeMode=CanResize`
3. **层级控制**：迷你窗口和截图浮动窗都Topmost，但通过创建顺序和`SetWindowPos` API确保截图在最上层
4. **状态同步**：TodoStore和ChatManager已经是单例，迷你窗口和全屏工作台都订阅变更事件自动刷新
5. **托盘菜单精简**：
   - 截图
   - 粘贴
   - ──────
   - AI 助手（打开全屏工作台→聊天Tab）
   - 待办（打开全屏工作台→待办Tab）
   - ──────
   - 设置（打开全屏工作台→设置Tab）
   - 退出
