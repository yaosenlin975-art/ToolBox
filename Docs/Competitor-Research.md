# ToolBox 竞品调研报告

> 调研时间：2026-07-13 | 调研人：产品经理 许清楚
> 调研目标：分析同赛道桌面工具的核心功能与差异化能力，提炼可移植到 ToolBox 的功能点

---

## 一、调研方法

### 1.1 调研范围

本次调研覆盖四大赛道共 14 款产品，选取原则为：市场口碑好、用户基数大、与 ToolBox 功能有交集或互补关系。

| 赛道 | 竞品 | 调研重点 |
|------|------|---------|
| 截图贴图类 | Snipaste、PixPin、ShareX、Snagit、Greenshot、Setuna | 截图/贴图/标注/OCR/录屏核心能力 |
| AI 助手/效率启动器类 | Raycast、Quicker、uTools、PowerToys | 快捷启动/AI 集成/插件生态/工作流自动化 |
| 待办/笔记类 | TickTick（滴答清单）、Flomo（浮墨笔记） | 待办管理/快速记录/日历视图/习惯打卡 |
| 截图管理/素材库类 | Eagle、CleanShot X | 截图历史管理/素材标注/云分享/标签体系 |

### 1.2 调研手段

- **WebSearch 检索**：搜索各竞品 2024-2026 年最新动态、功能更新日志、用户评测
- **官网信息采集**：逐一访问竞品官网，获取功能列表、定价模式、更新日志
- **第三方评测对比**：查阅少数派、什么值得买、掘金等平台的横向评测文章
- **移植可行性分析**：结合 ToolBox 现有架构（.NET 8 WPF）评估移植难度

### 1.3 评估维度

每个竞品从以下 5 个维度分析：
1. **产品定位** — 核心用户群与使用场景
2. **核心功能** — 功能清单与能力边界
3. **差异化亮点** — 与同类产品的区分点
4. **可移植到 ToolBox 的功能点** — 具体功能建议
5. **移植难度评估** — 低/中/高（基于 ToolBox 现有架构判断）

## 二、竞品逐个分析

### 2.1 Snipaste

| 维度 | 内容 |
|------|------|
| **产品定位** | 跨平台（Windows/Mac/Linux）专业截图贴图工具，国内数百万用户，主打"轻量、无广告、高流畅" |
| **核心功能** | 精准截图（自动检测窗口边缘、像素级捕捉）、贴图置顶、取色器、多屏适配、标注工具（马赛克/模糊/箭头/文字）、全局热键 |
| **差异化亮点** | ① 贴图功能行业标杆，支持鼠标穿透、透明度调节、多贴图叠加；② 专业版支持虚拟桌面、锁定纵横比、命令行静默截图；③ 跨三平台一致性体验；④ 超级截图（Super Snip）能力 |
| **可移植功能点** | ① 取色器功能（ToolBox 当前缺少）；② 截图锁定纵横比；③ 贴图鼠标穿透开关的文本提示反馈；④ 命令行静默截图到指定路径（支持自动化场景） |
| **移植难度** | 低 — 取色器为独立功能模块；命令行参数解析为标准 .NET 能力 |

### 2.2 PixPin

| 维度 | 内容 |
|------|------|
| **产品定位** | 国产新一代免费截图工具，"截图+贴图+OCR+GIF 录制"全能整合，被称为"截图界新卷王" |
| **核心功能** | 智能截图（区域/窗口/全屏/滚动长截图）、桌面贴图、离线 OCR 文字识别（支持中英日韩 10+ 语言）、一键翻译、GIF/视频录屏、标注工具（箭头/马赛克/序号/高亮）、贴图自动文字识别 |
| **差异化亮点** | ① **离线 OCR 毫秒级识别**，无需联网/API Key，贴图后可直接选中文本复制；② 原生长截图（自动滚动拼接，横向纵向均支持）；③ 内存占用仅 ~15MB，运行极快；④ 截图历史按时间轴保存，支持关键词搜索 |
| **可移植功能点** | ① **离线 OCR**（最高优先级，ToolBox 当前完全缺失）；② 长截图/滚动截图；③ 贴图上直接选取文字（OCR+文本选择交互）；④ 截图翻译（选中文本一键翻译） |
| **移植难度** | 中 — OCR 需集成本地模型（如 Tesseract 或 PaddleOCR .NET 绑定），长截图需要滚动捕获逻辑；贴图文字选择需要改造 ScrapWindow |

### 2.3 ShareX

| 维度 | 内容 |
|------|------|
| **产品定位** | Windows 平台最强开源截图录屏工具（GPL-3.0），极客/技术博主首选，强调自动化工作流 |
| **核心功能** | 10+ 截图模式（全屏/窗口/矩形/多区域/手绘/滚动/网页/OCR/自动截图）、GIF/MP4 录屏、80+ 云存储自动上传、自动化工作流（截图→特效→上传→短链→复制）、内置图片编辑器、颜色选择器、QR 码识别、哈希校验 |
| **差异化亮点** | ① **After-Capture/After-Upload 自动化工作流**，一次配置永久生效；② 80+ 云端上传目标；③ 多区域截图（不同形状）；④ 2025 年升级 .NET 9，历史记录迁移到 SQLite 数据库；⑤ 命令行参数支持 Pin/OCR/QRCode 等操作 |
| **可移植功能点** | ① **截图后自动化动作链**（截图→OCR→复制/保存/上传可配置）；② 截图历史 SQLite 存储 + 标签 + 收藏 + 重命名；③ 多区域截图（选多个不连续区域一次性截取）；④ QR 码识别；⑤ 自动截图定时器（ToolBox 已有 Cron 但可参考 ShareX 的多模式） |
| **移植难度** | 中 — 自动化动作链需设计配置 UI 和执行引擎；SQLite 历史存储 ToolBox 已有 MemoryStore 可参考；QR 码识别可用 ZXing.Net |

### 2.6 Setuna（ToolBox 前身参考）

| 维度 | 内容 |
|------|------|
| **产品定位** | 轻量级截图贴图工具（单文件 <500KB），面向程序员/UI 设计师/测试工程师，专注"截图-比对-归档"闭环 |
| **核心功能** | 全屏/窗口/区域截图、桌面贴图（悬浮置顶/拖拽/缩放/透明度调节）、截图回收站、热键设置、参考图列表、多 DPI 显示器支持、开机自启、重启恢复贴图 |

### 2.12 Flomo（浮墨笔记）

| 维度 | 内容 |
|------|------|
| **产品定位** | 卡片式笔记工具，"像发微博一样记录"，聚焦碎片灵感捕捉与知识回顾 |
| **核心功能** | 快速记录（极简输入框/微信输入/语音转文字）、多级标签系统（#/子标签）、每日回顾（随机推送历史卡片）、热力图（可视化记录习惯）、AI Agent（读过所有笔记的 AI，可对话回顾/思考/提醒）、MCP 连接（Claude/ChatGPT 中直接读写）、笔记关系图 |
| **差异化亮点** | ① **无压记录**——极简设计，仅支持基本格式，拒绝排版干扰；② **AI Agent 读全部笔记**——真正基于用户全部数据的个性化 AI；③ 随机漫步回顾——从一条笔记游走到另一条；④ 冷门笔记自动推送（30 天未打开提醒）；⑤ 认知地图（笔记自动涌现主题山峰） |

### 3.3 P0 紧急功能详细说明

#### P0-1：离线 OCR 文字识别

- **来源**：PixPin（离线 OCR 是其核心杀手锏）、ShareX（Tesseract 集成）
- **场景**：截取报错弹窗提取代码、截取视频字幕、截取不可复制的网页文本、截取 PDF/图片中的文字
- **建议方案**：集成 Tesseract OCR 或 PaddleOCR .NET 绑定，在截图工具栏增加"OCR"按钮，识别结果可编辑/复制/翻译/发送到 AI 助手
- **预期效果**：填补 ToolBox 截图能力最大空白，将截图从"视觉捕获"升级为"信息提取"

#### P0-2：剪贴板历史管理

- **来源**：Raycast（可视/可搜/无限历史）
- **场景**：复制多段文本后需要回溯、复制的图片需要再次粘贴、跨应用复制粘贴中转
- **建议方案**：后台监听剪贴板链，存储文本/图片/文件历史，在工作台增加"剪贴板"页面或迷你窗口增加入口，支持搜索/置顶/收藏
- **预期效果**：补齐全局效率工具的基础能力，与截图（复制到剪贴板）和 AI 助手（发送剪贴板内容）形成协同

#### P0-3：屏幕取色器

- **来源**：Snipaste、PowerToys ColorPicker
- **场景**：前端开发取色、设计参考色值提取、UI 调试
- **建议方案**：全局热键（如 Ctrl+Shift+C）唤起取色器，鼠标移动实时显示放大镜+像素色值（HEX/RGB/HSL），点击复制到剪贴板
- **预期效果**：补齐截图工具标配能力，低开发成本高用户感知

## 五、结论与建议

### 5.1 ToolBox 当前能力定位

ToolBox 在截图贴图赛道有扎实基础（贴图置顶、回收站、透明度调节、多屏支持均对标 Setuna/Snipaste），LLM 助手+待办+定时日报的组合在同类工具中具有独特性。但在以下方面存在明显短板：

1. **截图能力缺失**：无 OCR、无长截图、无取色器、无录屏——这些是 2024-2026 年截图工具的标配能力
2. **效率工具基础能力缺失**：无剪贴板历史、无代码片段、无快速启动——作为"桌面助手"定位但缺少效率工具标配
3. **待办管理停留在基础**：无日历视图、无智能日期识别、子任务/进度仍为设计目标——与 TickTick 差距明显
4. **截图管理深度不足**：截图历史仅按时间筛选，无标签/分类/标注/评分——与 Eagle/CleanShot X 的素材管理理念差距大

### 5.2 短期建议（P0，1-2 个迭代）

**聚焦"补齐截图工具标配能力"**，优先实现：
1. **离线 OCR** — 投入产出比最高，是用户从 Snipaste/PixPin 迁移的必要条件
2. **屏幕取色器** — 开发成本低，用户感知强
3. **剪贴板历史** — 补齐效率工具基础能力，与截图/AI 协同

### 5.3 中期建议（P1，2-4 个迭代）

**聚焦"截图工作流深化 + 待办体验提升"**：
4. **截图后自动化动作链** — 截图→OCR→复制/保存/发送 AI 可配置
5. **长截图/滚动截图** — 高频需求
6. **智能日期识别** — 降低待办创建摩擦
7. **截图标签与智能分类** — 截图历史从时间线升级为素材库
8. **代码片段管理** — 效率用户刚需
9. **截图后悬浮预览卡** — 减少截图后操作步骤

### 5.4 长期建议（P2-P3，4+ 个迭代）

**聚焦"差异化能力与生态构建"**：
10. 待办日历视图 + 番茄钟
11. 上下文感知快捷操作
12. AI 基于全数据个性化（LLM 读取待办/截图/会话历史）
13. 步骤捕获 / 智能打码等高级截图能力
14. 插件化扩展架构（参考 uTools/Raycast 生态模式）

### 5.5 差异化战略建议

ToolBox 的独特优势在于**"截图贴图 + LLM 助手 + 待办管理 + 定时日报"的四合一整合**，这是任何单一竞品都不具备的。建议在补齐基础能力的同时，强化这四个模块的**协同闭环**：

- **截图 → AI**：截图后一键发送到 AI 助手分析（当前仅能贴图）
- **AI → 待办**：AI 对话中自动提取待办（已有 TodoTools，可增强智能提取）
- **待办 → 日报**：待办完成情况自动汇入日报（当前日报仅基于截图）
- **截图 → 待办**：截图标注关联待办（截图作为待办的视觉上下文）

这种"截图为眼、AI 为脑、待办为手、日报为镜"的闭环，是 ToolBox 对抗垂直竞品的核心壁垒。
