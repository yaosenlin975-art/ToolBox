# 长截图/滚动截图 产品规格说明

- **优先级**：P1
- **对应设计**：`Docs/design/features/05-Scrolling-Screenshot.md`
- **版本**：v1.0

---

## 一、功能概述

为 ToolBox 增加长截图（滚动截图）能力：用户在截图模式中选择一个窗口或区域后，ToolBox 自动向目标发送滚动指令，逐帧截取并拼接为一张完整的长图。支持垂直和水平两种滚动方向，适用于网页长文章、聊天记录、代码文件、Excel 表格等需要超出屏幕范围截取的场景。截取完成后进入预览编辑窗口，可裁剪拼接区域后保存或贴图。

## 二、用户故事

| # | 用户故事 | 验收优先级 |
|---|---------|-----------|
| US1 | 作为**信息收集者**，我想要一次性截取整个网页长文章，以便离线保存或分享给同事而不需要分段截图 | P1 |
| US2 | 作为**开发者**，我想要截取一长段代码或终端输出，以便在 Code Review 或 Bug Report 中附上完整上下文 | P1 |
| US3 | 作为**聊天用户**，我想要截取完整的微信/钉钉聊天记录，以便归档重要的工作沟通内容 | P1 |
| US4 | 作为**表格用户**，我想要横向滚动截取宽表格（如 Excel），以便在一个画面中看到所有列 | P2 |

## 三、验收标准

### 入口与选区

- [ ] **AC1.1**：按下 `Ctrl+Shift+L` 直接进入长截图模式；或在普通截图选区后工具栏选择"长截图"
- [ ] **AC1.2**：长截图模式下，CaptureWindow 顶部显示"📜 长截图模式"标签，引导文字"选择窗口或区域后自动开始滚动截取 / ESC 取消"
- [ ] **AC1.3**：用户可通过选项切换滚动方向：垂直（默认）/ 水平

### 自动截取

- [ ] **AC2.1**：选区确认后自动开始滚动截取，每次滚动后等待 300ms（默认可配）再截下一帧，确保内容渲染完成
- [ ] **AC2.2**：截取过程中，选区底部显示进度条 + 帧计数"第 N 帧"；实时显示最新帧缩略图（120px 宽）浮在进度条上方
- [ ] **AC2.3**：自动检测页面底部：连续 2 帧内容无变化 → 自动停止截取
- [ ] **AC2.4**：用户随时可按 ESC 手动停止截取（已截取的帧保留）
- [ ] **AC2.5**：最大截取帧数默认 50 帧，超出自动停止（可在设置中调整）

### 拼接与预览

- [ ] **AC3.1**：截取完成后自动进入预览编辑窗口（ScrollingCapturePreview，最大 1200×800px），显示拼接后的完整长图
- [ ] **AC3.2**：长图中以半透明红线标记各帧拼接边界，用户可拖拽边界线调整拼接位置
- [ ] **AC3.3**：拼接接缝处无明显视觉断裂（相邻帧重叠区域做线性渐隐融合）
- [ ] **AC3.4**：自动裁剪首帧顶部非内容区（如浏览器标签栏）和末帧底部纯色空白
- [ ] **AC3.5**：预览窗口支持 Ctrl+滚轮缩放，工具栏提供"保存 PNG""复制到剪贴板""贴图置顶""裁剪""撤销"按钮

### 滚动兼容

- [ ] **AC4.1**：提供三种滚动模式切换：鼠标滚轮（默认）/ PageDown 按键 / 箭头键，适配不同应用的滚动行为
- [ ] **AC4.2**：若目标窗口最小化或不可见，提示"请先显示目标窗口"

## 四、功能范围

### In Scope（本期实现）

- 垂直长截图（热键 `Ctrl+Shift+L`）
- 水平长截图
- 窗口/区域两种选区方式
- 自动滚动 + 逐帧截取 + 图像拼接
- 自动停止检测（连续相同帧）
- 手动 ESC 停止
- 预览编辑窗口（拼接边界调整、裁剪、缩放）
- 三种滚动模式（鼠标滚轮/PageDown/箭头键）
- 保存 PNG / 复制到剪贴板 / 贴图置顶

### Out of Scope（本期不实现）

- GIF/视频滚动录制
- 网页自动全文截取（无需滚动，直接渲染完整页面）
- 拼接精度手动像素级微调
- 批量长截图
- 定时自动长截图

## 五、交互流程

```
1. 用户按 Ctrl+Shift+L → 进入长截图模式
2. CaptureWindow 显示，顶部标注"📜 长截图模式"
3. 用户选择目标窗口（点击窗口或拖拽选区）
4. 选区确认 → 自动开始：
   a. 截取第 1 帧 → SendMessage 发送滚轮事件 → 等待 300ms
   b. 截取第 2 帧 → ScrollDetector 对比与第 1 帧差异
   c. 循环至检测到底部或达最大帧数或用户按 ESC
5. ImageStitcher 拼接所有帧（特征匹配 + 渐隐融合）
6. ScrollingCapturePreview 显示结果
7. 用户调整拼接边界 → 点击"保存 PNG"或"贴图置顶"
```

## 六、非功能需求

### 性能

- 单帧截取 + 压缩 ≤ 200ms（1920×1080 窗口）
- 50 帧拼接总耗时 ≤ 5 秒
- 拼接期间 UI 不阻塞（后台线程执行）
- 截取过程中内存峰值 ≤ 500MB（50 帧 JPEG Quality=80 压缩存储）

### 安全

- 完全本地处理，不涉及网络
- 滚动模拟仅向用户明确选择的目标窗口发送消息

### 可访问性

- 进度条 + 帧计数双重反馈
- ESC 随时可停止

## 七、依赖项

| 依赖 | 类型 | 说明 |
|------|------|------|
| `user32.dll` SendMessage | 系统 API | WM_MOUSEWHEEL / WM_VSCROLL 滚动模拟 |
| `Graphics.CopyFromScreen` | .NET 内置 | 窗口区域截取 |
| `CaptureWindow` | 现有模块 | 复用选区逻辑，增加长截图模式 |
| `HotkeyManager` | 现有模块 | 注册 `Ctrl+Shift+L` 热键 |
| `ScrollingCaptureEngine` | 新建模块 | 滚动控制 + 逐帧捕获 |
| `ImageStitcher` | 新建模块 | 图像拼接算法 |
| `ScrollDetector` | 新建模块 | 滚动停止检测 |
| `CacheManager` | 现有模块 | 长截图结果存入缓存 |

## 八、竞品参考

| 竞品 | 实现方式 | ToolBox 借鉴点 |
|------|---------|---------------|
| **PixPin** | 原生长截图，自动滚动拼接，横向纵向均支持 | 自动滚动 + 拼接 + 预览编辑的完整闭环 |
| **ShareX** | 滚动截图支持多种窗口，自动拼接 | 多种滚动模式切换（滚轮/按键）应对不同应用 |
| **Snagit** | Scrolling Capture 支持全景和水平滚动，拼接精度高 | 帧边界可拖拽调整 + 自动裁剪非内容区的预览编辑 |

ToolBox 差异化：(1) 与贴图系统联动——长截图结果可直接贴图置顶；(2) 与截图历史集成——长截图自动进入 HistoryView；(3) 轻量拼接算法——无 OpenCV 依赖，纯 C# 实现。
