# 截图后悬浮预览卡 产品规格说明

- **优先级**：P1
- **对应设计**：`Docs/design/features/09-Preview-Card.md`
- **版本**：v1.0

---

## 一、功能概述

截图完成后，在屏幕右下角显示一个悬浮预览卡片，展示截图缩略图和快捷操作按钮。用户可将预览卡拖拽到目标应用（如聊天窗口、文档编辑器）直接插入图片；点击卡片上的按钮快速编辑、贴图或复制。预览卡 5 秒后自动淡出消失，鼠标悬停时保持显示。该功能减少了截图后的操作步骤，对标 CleanShot X 的悬浮预览交互。

## 二、用户故事

| # | 用户故事 | 验收优先级 |
|---|---------|-----------|
| US1 | 作为**聊天用户**，我想要截图后直接将右下角的缩略图拖拽到微信/钉钉聊天窗口，以便省去"复制 → 切窗口 → 粘贴"的步骤 | P1 |
| US2 | 作为**文档编辑者**，我想要截图后快速确认截图内容是否正确（通过缩览图预览），以便决定是保留、重截还是编辑 | P1 |
| US3 | 作为**贴图重度用户**，我想要截图后一键点击"贴图"按钮将截图置顶，以便快速进入比对模式 | P1 |
| US4 | 作为**注重效率的用户**，我想要在截图用于贴图时跳过预览卡（自动贴图），以便不被不必要的卡片打断 | P2 |

## 三、验收标准

### 预览卡显示

- [ ] **AC1.1**：截图完成后，屏幕右下角滑入 PreviewCardWindow（200×150px, Topmost），入场动画 300ms ease-out（TranslateTransform Y 150→0 + Opacity 0→1）
- [ ] **AC1.2**：预览卡背景为 BgElevatedBrush 95% 不透明度，圆角 14px，外阴影 ShadowFloat
- [ ] **AC1.3**：缩略图区域（184×100px）以 Uniform 模式展示截图，背景棋盘格图案表示透明区域
- [ ] **AC1.4**：缩略图下方显示三个操作按钮：✏️ 编辑 / 📌 贴图 / 📋 复制（IconButton 变体 24×24）

### 自动消失与悬停

- [ ] **AC2.1**：预览卡 5 秒后自动执行淡出动画（Opacity 1→0, 300ms）然后关闭
- [ ] **AC2.2**：鼠标悬停在预览卡上时，自动消失计时重置（卡片保持显示），鼠标移开后重新开始 5 秒计时
- [ ] **AC2.3**：鼠标悬停时卡片轻微放大（ScaleTransform 1.0→1.05, 150ms ease-out）

### 拖拽分享

- [ ] **AC3.1**：按住预览卡上的缩略图可拖拽到其他应用窗口，释放时将截图粘贴/插入
- [ ] **AC3.2**：拖拽 DataObject 包含多种格式（Bitmap / DIB / FileDrop / PNG 流），兼容主流应用
- [ ] **AC3.3**：拖拽结束后预览卡自动关闭

### 快捷操作

- [ ] **AC4.1**：点击"✏️ 编辑"→ 打开标注编辑器（现有 Paint 界面）
- [ ] **AC4.2**：点击"📌 贴图"→ 创建 ScrapWindow 贴图置顶
- [ ] **AC4.3**：点击"📋 复制"→ 截图复制到剪贴板，Toast 提示"已复制到剪贴板"
- [ ] **AC4.4**：右键预览卡弹出菜单：发送到 AI / 保存为文件 / 丢弃

### 配置

- [ ] **AC5.1**：设置页"📸 截图"Section 新增开关"☑ 截图后显示悬浮预览卡"（默认开启）
- [ ] **AC5.2**：可配置预览卡显示时长（3/5/10 秒）
- [ ] **AC5.3**：可配置"贴图模式下跳过预览卡"（当用户在截图中选择贴图模式时不显示预览卡）

## 四、功能范围

### In Scope（本期实现）

- 截图完成 → 右下角悬浮预览卡（入场动画）
- 缩略图预览
- 三个快捷操作按钮（编辑/贴图/复制）
- 右键菜单（发送到 AI / 保存 / 丢弃）
- 拖拽分享到其他应用
- 5 秒自动淡出 + 悬停重置计时器
- 设置开关和参数配置
- 多显示器支持（显示在截图所在屏幕的右下角）

### Out of Scope（本期不实现）

- 预览卡堆叠（连续多次截图时多个预览卡排队显示）
- 预览卡内直接标注
- 预览卡历史记录
- 预览卡拖拽到浏览器 web 应用的精确兼容

## 五、交互流程

### 标准流程

```
1. 用户完成截图 → CaptureWindow.OnCaptureComplete(bitmap)
2. PreviewCardManager.Show(bitmap) → 创建 PreviewCardWindow
3. 窗口定位：截图所在屏幕右下角（考虑任务栏高度），ShowActivated=False
4. 入场动画播放（300ms 滑入 + 淡入）
5. 用户查看缩略图确认内容 OK
6. 5 秒后无交互 → 淡出动画（300ms）→ 窗口关闭
```

### 悬停 + 操作流程

```
1. 预览卡显示中（倒计时还剩 3 秒）
2. 用户鼠标移入预览卡 → 倒计时重置为 5 秒 → 卡片微微放大
3. 用户点击"📋 复制"按钮 → 截图复制到剪贴板
4. Toast 提示"已复制到剪贴板" → 预览卡立即关闭
```

### 拖拽流程

```
1. 用户按住预览卡上的缩略图
2. 拖拽到微信聊天窗口 → 释放
3. 截图图片插入聊天输入框
4. 预览卡自动关闭
```

## 六、非功能需求

### 性能

- 预览卡从截图完成到显示 ≤ 200ms
- 入场动画 60fps（WPF Storyboard + DoubleAnimation）
- 缩略图生成 ≤ 50ms（184×100 缩放）

### 安全

- 拖拽 DataObject 中包含的临时文件路径指向 CacheManager 管理的截图
- 预览卡不含截图敏感信息外传

### 可访问性

- 操作按钮支持键盘 Tab 切换 + Enter 触发
- 入场/离场动画时长可配（减少动画以适配偏好设置）

## 八、竞品参考

| 竞品 | 实现方式 | ToolBox 借鉴点 |
|------|---------|---------------|
| **CleanShot X** | 截图后左下角悬浮预览，拖拽分享/点击编辑，自动消失 | 核心交互——悬浮位置、拖拽分享、自动消失、悬停保持 |
| **Snipaste** | 截图后直接贴图（无预览卡） | ToolBox 通过预览卡在"确认内容"和"贴图"之间增加一个缓冲步骤 |
| **ShareX** | 截图后根据 After-Capture 配置直接执行 | 与 ToolBox 动作链协同——预览卡 + 动作链可并行 |

ToolBox 差异化：(1) 可与动作链共存——预览卡提供手动选择，动作链提供自动处理；(2) 贴图/编辑/复制三合一快速入口；(3) 拖拽分享的格式兼容（多种 DataObject 格式）。
