# 截图标签与智能分类 产品规格说明

- **优先级**：P1
- **对应设计**：`Docs/design/features/07-Screenshot-Tags.md`
- **版本**：v1.0

---

## 一、功能概述

为 ToolBox 截图历史增加标签管理和智能分类能力。用户可为每张截图手动添加标签（如 #bug、#设计稿、#会议纪要），系统支持基于 LLM 的自动标签推荐（分析截图内容后推荐 3-5 个标签）。截图历史视图增加标签云筛选、星级评分（1-5 星）和颜色筛选。该功能将截图历史从"纯时间线"升级为"可检索的素材库"，与 Eagle 的素材管理理念对齐。

## 三、验收标准

### 手动标签

- [ ] **AC1.1**：截图完成后，工具栏显示标签编辑区——已添加的标签以胶囊样式展示，支持输入新标签后 Enter/Tab 添加，点击胶囊 × 删除
- [ ] **AC1.2**：标签输入时实时匹配已有标签（在标签库中搜索），最多显示 5 个建议
- [ ] **AC1.3**：标签以 Token 胶囊样式呈现：`Border CornerRadius=999, Background=AccentSoftBrush, Padding=8,4`
- [ ] **AC1.4**：每个截图支持最多 20 个标签

### 自动打标（LLM）

- [ ] **AC2.1**：设置页"🏷 截图标签"Section 中可开启"LLM 自动推荐标签"（默认关闭）
- [ ] **AC2.2**：开启后，截图完成时自动调用 LLM 分析图像 → 推荐 3-5 个标签 → 显示在标签编辑区（标签带有"AI"标记以示区分）
- [ ] **AC2.3**：用户可逐个接受或删除 AI 推荐的标签
- [ ] **AC2.4**：AI 推荐标签优先使用已有标签池中的标签，减少标签膨胀

### 星级评分

- [ ] **AC3.1**：截图工具栏标签编辑区旁显示 5 颗星评分控件（16×16 星形图标）
- [ ] **AC3.2**：点击第 N 颗星 → 评分 = N；再次点击同位置 → 清除评分
- [ ] **AC3.3**：已评星星色为 AccentBrush，未评为 TextTertiaryBrush

### 标签与颜色筛选

- [ ] **AC4.1**：HistoryView 顶部增加标签云栏——水平排列标签胶囊（FilterTabStyle），按使用频次降序，每个标签显示计数如"bug 12"
- [ ] **AC4.2**：点击标签 → 筛选出含该标签的截图；支持多标签 AND 组合筛选
- [ ] **AC4.3**：标签云旁增加星级筛选（⭐1 ⭐2 ⭐3 ⭐4 ⭐5，FilterTabStyle）
- [ ] **AC4.4**：自动提取截图主色调（3 个），在筛选栏以色块展示；点击色块按色相 ±30° 筛选
- [ ] **AC4.5**：支持标签 + 评分 + 颜色 + 时间的组合查询

### 截图详情面板

- [ ] **AC5.1**：HistoryView 右侧详情面板扩展：标签区、星级评分、备注编辑区、主色调色块
- [ ] **AC5.2**：详情面板中可直接编辑标签、修改评分、添加备注

### LLM 工具

- [ ] **AC6.1**：AI 对话中可调用 `search_screenshots` 按标签/评分/关键词搜索截图
- [ ] **AC6.2**：AI 对话中可调用 `tag_screenshot` 为最近截图添加标签

## 四、功能范围

### In Scope（本期实现）

- 手动标签（添加/删除/建议/搜索已有标签）
- 5 星评分
- 主色调自动提取（简化 K-Means，纯 C# 实现）
- 标签云 + 星级 + 颜色筛选栏
- 组合查询（标签 AND 评分 AND 时间范围）
- LLM 自动标签推荐（可选，默认关闭）
- 截图详情面板扩展
- LLM 工具 `search_screenshots` / `tag_screenshot`
- 标签持久化（`screenshot_tags.json`）

### Out of Scope（本期不实现）

- 智能文件夹（多条件自动分类规则动态生成虚拟文件夹）
- 标签层级/嵌套（本次仅扁平结构）
- 批量自动打标历史截图
- 标签导入导出
- 人脸/物体识别自动打标
- OCR 关键词自动转标签

## 五、交互流程

### 手动标签流程

```
1. 用户截图完成 → 工具栏出现标签编辑区
2. 用户输入"bug" → Enter → 生成标签胶囊 [bug ✕]
3. 用户继续输入"登录" → 下拉出现已有标签"登录页"→ 点击选择
4. 用户点击四颗星 → 评分 = 4
5. 截图保存 → 元数据（标签 + 评分 + 主色调）写入 ScreenshotTagStore
```

### 筛选流程

```
1. 用户在工作台 screenshots 页
2. 点击标签云中的"bug 12"→ 筛选出 12 张含 bug 标签的截图
3. 再点击 ⭐4 → 进一步筛选出评分 ≥ 4 的 bug 截图
4. 点击红色色块 → 进一步筛选出红色系截图
5. 结果列表显示 3 张同时满足所有条件的截图
```

## 六、非功能需求

### 性能

- 标签云渲染 ≤ 50ms（100 个标签以内）
- 颜色提取 ≤ 500ms/张（后台异步执行）
- 组合查询 ≤ 100ms（500 张截图以内）
- LLM 自动打标超时 10 秒

### 安全

- 标签数据本地存储
- LLM 自动打标仅发送缩略图 base64（200×200），不发送原图
- 用户确认后 AI 标签才生效

### 可访问性

- 星级评分支持键盘操作（← → 调整，0 清除）
- 标签胶囊的删除按钮可 Tab 聚焦

## 八、竞品参考

| 竞品 | 实现方式 | ToolBox 借鉴点 |
|------|---------|---------------|
| **Eagle** | 标签系统 + 智能文件夹 + 颜色筛选 + 星级评分 | 标签云 + 颜色筛选 + 星级评分的多维筛选体系 |
| **ShareX** | 截图历史 SQLite 存储 + 标签 + 收藏 + 重命名 | 标签 + 收藏 + 时间组合筛选的查询模式 |
| **CleanShot X** | 截图后快速标注和分享 | 截图完成工具栏的标签编辑区交互 |

ToolBox 差异化：(1) LLM 自动打标——与 Eagle 的纯手动标签不同，AI 主动推荐减少用户操作；(2) 标签 + LLM 工具联动——AI 对话中可按标签搜索截图；(3) 轻量实现——纯 C# 颜色提取，无 OpenCV 依赖。
