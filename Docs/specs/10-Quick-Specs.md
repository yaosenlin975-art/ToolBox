# P2+P3 功能简要规格说明

- **版本**：v1.0
- **对应设计**：`Docs/design/features/10-Quick-Designs.md`
- **说明**：以下 11 个功能为中期（P2）和远期（P3）优先级，每项提供一句话概述、核心用户故事、验收标准。

---

## P2-01：日历视图

**概述**：为待办管理增加月/周/日日历视图，将 TodoItem 的 DueDate 映射到日历网格中，支持拖拽待办调整截止日期，弥补当前仅有列表视图的不足。

**核心用户故事**：
- 作为**时间管理者**，我想要在月历中看到哪天有待办截止，以便直观规划一周的工作负载
- 作为**计划用户**，我想要拖拽列表中的待办到日历某一天，以便快速调整截止日期（Timeboxing）

**验收标准**：
- [ ] TodoView 增加"列表/日历"视图切换按钮，日历视图基于自定义 WPF UniformGrid 月历控件
- [ ] 月历中每个日期格子显示当日待办数量徽标，有截止日期的格子以 AccentSoftBrush 标记
- [ ] 支持从待办列表拖拽条目到日历日期格子 → DueDate 自动更新
- [ ] 支持月/周/日三种视图切换
- [ ] 点击日历日期 → 显示当日待办列表浮层

## P2-02：上下文感知快捷操作

**概述**：根据用户当前活动窗口和选中内容，在迷你窗口或浮动面板中智能推荐相关操作。例如在 VS Code 中选中代码 → 推荐"发送到 AI"/"保存为片段"。

**核心用户故事**：
- 作为**多应用用户**，我想要在不同应用中看到不同的推荐操作，以便不需要记住快捷键也能快速调用 ToolBox 功能
- 作为**开发者**，我想要选中代码后一键发送到 AI 助手分析，以便无需手动复制粘贴

**验收标准**：
- [ ] 通过 `GetForegroundWindow` + 进程名识别当前应用（浏览器/IDE/Office 等 10+ 种）
- [ ] 迷你窗口增加上下文推荐区，显示 3-5 个推荐操作图标按钮
- [ ] 选中文本后长按右键或热键唤起上下文菜单，显示推荐操作
- [ ] 推荐规则通过 JSON 配置文件管理，可自定义扩展
- [ ] 获取选中文本时临时备份剪贴板内容，获取后恢复

## P2-03：AI 基于全数据个性化

**概述**：将 LLM 助手与用户全量本地数据深度整合——AI 对话时可访问待办列表、截图历史（含标签/OCR 文字）、代码片段、剪贴板历史、定时日报。系统提示词动态注入近期用户活动摘要。

**核心用户故事**：
- 作为**AI 助手用户**，我想要问"我上周三截了什么图？"时 AI 能查询截图历史并回答，以便真正把 AI 当个人助理
- 作为**日报用户**，我想要 AI 基于最近的待办完成情况和截图活动自动生成日报摘要

**验收标准**：
- [ ] `PersonalizedPromptBuilder` 扩展 `ISystemPromptBuilder`，在系统提示词中注入近期活动摘要
- [ ] AI 回答"最近截了什么图"时能调用 `search_screenshots` 返回结果
- [ ] 各 Store（TodoStore/ScreenshotTagStore/ClipboardStore/SnippetStore）提供轻量摘要接口
- [ ] 定期检测冷门内容（30 天未查看的旧截图/待办）→ 迷你窗口推送回顾提醒
- [ ] 个性化功能默认关闭，在设置中手动开启

## P2-04：步骤捕获

**概述**：启动"步骤捕获模式"后监听用户鼠标点击，自动截取点击前后的屏幕画面，生成带序号标注的步骤教程截图。适用于软件操作文档和 Bug 复现步骤记录。

**核心用户故事**：
- 作为**技术文档编写者**，我想要自动记录操作步骤截图而不用手动一次次截图，以便快速制作软件使用教程
- 作为**QA 测试工程师**，我想要自动记录 Bug 复现的每一步操作画面，以便在 Bug Report 中提供精确步骤

**验收标准**：
- [ ] 热键唤起步骤捕获模式 → 全局鼠标钩子 WH_MOUSE_LL 检测每次点击
- [ ] 每次点击前后 200ms 各截一帧，自动叠加步骤序号（圆形编号，右下角）
- [ ] 点击位置以红色半透明圆圈高亮（200ms 脉冲后消失）
- [ ] 停止捕获后进入预览窗口，显示所有步骤帧序列
- [ ] 支持导出为多张 PNG 或一张拼接长图

## P2-05：智能打码

**概述**：截图分享前自动检测并遮挡敏感信息——邮箱、电话、IP、信用卡号、人脸、二维码。打码方式支持模糊和马赛克两种。用户可手动添加/调整打码区域。

**核心用户故事**：
- 作为**隐私敏感用户**，我想要在分享截图前自动遮挡其中的手机号和邮箱，以便安全地将截图发到公共频道
- 作为**社交媒体用户**，我想要给截图中的头像/人脸自动打码，以便保护他人隐私

**验收标准**：
- [ ] 在截图标注工具栏增加"🔒 智能打码"按钮
- [ ] 点击后自动检测：邮箱（正则）、电话号码（正则）、IP（正则）、二维码（ZXing.Net）
- [ ] 检测到的区域以半透明遮罩标记，用户可逐个确认/取消/调整区域
- [ ] 打码方式可选：高斯模糊 / 马赛克（像素化）
- [ ] 人脸检测可选集成 OpenCvSharp4（Haar Cascade），无 OpenCV 时仅做文本检测

## P2-06：番茄钟

**概述**：内置番茄钟计时器，25 分钟专注 + 5 分钟休息循环。可关联待办，迷你窗口显示倒计时圆环。专注完成后自动记录并统计当日番茄数，与日报联动。

**核心用户故事**：
- 作为**专注工作者**，我想要开启 25 分钟番茄钟并关联正在做的待办，以便专注期间不被其他事务打断
- 作为**日报用户**，我想要在日报中看到当日完成了几次番茄专注，以便量化自己的深度工作时间

**验收标准**：
- [ ] 迷你窗口增加"番茄钟模式"——点击番茄图标切换，显示圆形倒计时进度环
- [ ] 开始时可选择关联一条待办（可选）
- [ ] 专注完成 → 迷你窗口提示 + 自动记录到 `pomodoro_log.json`
- [ ] 支持自定义时长（默认 25 分钟专注 / 5 分钟休息）
- [ ] 迷你窗口底部状态栏显示"🍅×3"当日番茄数

## P3-01：屏幕冻结模式

**概述**：一键冻结当前屏幕画面——截取全屏作为静态图片覆盖显示，用户可从容选择截图区域。解决捕获瞬态 UI（下拉菜单、tooltip、hover 效果）时按热键导致 UI 消失的痛点。

**核心用户故事**：
- 作为**UI 测试者**，我想要冻结包含下拉菜单的画面后再截图，以便精确捕获菜单样式和内容

**验收标准**：
- [ ] 热键唤起屏幕冻结 → 全屏截取 → 各显示器创建全屏静态覆盖窗口
- [ ] 覆盖窗口上叠加 CaptureWindow 选区控件，用户可正常选区截图
- [ ] ESC 或截图完成后自动解除冻结
- [ ] 多显示器各自创建独立冻结窗口

## P3-02：QR 码识别

**概述**：截图后可识别其中的二维码/条形码，解析结果可直接打开 URL 或复制文本。NuGet ZXing.Net 实现，纯 .NET 无额外依赖。

**核心用户故事**：
- 作为**移动端用户**，我想要截取电脑屏幕上的二维码后用 ToolBox 直接识别 URL，以便在电脑上打开链接而不是用手机扫码

**验收标准**：
- [ ] 截图工具栏增加"QR"按钮 → 对截图执行 ZXing 解码
- [ ] 识别到 URL → 弹出"打开链接 / 复制链接"选项
- [ ] 识别到文本 → 直接显示文本 + 复制按钮
- [ ] 支持 QR Code / Data Matrix / Code 128 / EAN-13
- [ ] 支持从剪贴板图片和截图历史图片识别

## P3-03：窗口置顶快捷键

**概述**：任意窗口一键置顶/取消置顶（`Ctrl+Shift+T`）。补充 ToolBox 仅能置顶贴图窗口的限制，适用于需要对比参考的应用场景。

**核心用户故事**：
- 作为**多任务用户**，我想要将计算器窗口置顶以便在做表格时随时参考，而不是来回切换窗口

**验收标准**：
- [ ] 按 `Ctrl+Shift+T` → 当前前台窗口切换 Topmost 状态
- [ ] 置顶窗口边框短暂闪烁提示（AccentBrush, 300ms）
- [ ] 再次按同热键 → 取消置顶
- [ ] 热键可在设置中自定义

## P3-04：习惯打卡

**概述**：在待办系统基础上增加习惯追踪——创建每日/每周习惯，完成后打卡。用热力图（GitHub 绿格子风格）可视化打卡记录。与可重复待办协同。

**核心用户故事**：
- 作为**习惯养成者**，我想要每天打卡"运动 30 分钟"并用热力图看坚持情况，以便激励自己保持习惯

**验收标准**：
- [ ] 在 TodoView 增加"习惯打卡"子页面 → 显示习惯列表 + 今日打卡按钮
- [ ] 创建习惯：名称 / 频率（每日/每周）/ 目标次数
- [ ] 热力图可视化：7 列 × N 行色块（类似 GitHub contribution graph）
- [ ] 连续打卡天数（Streak）显示在习惯卡片上
- [ ] 习惯可关联一条每日待办模板（复用 RepeatConfig）

## P3-05：云分享/图床上传

**概述**：截图后可一键上传到图床获取分享链接并复制到剪贴板。支持配置多个上传目标（Imgur、自定义 WebDAV 等）。与截图动作链集成（"截图→上传→复制链接"）。

**核心用户故事**：
- 作为**社区用户**，我想要截图后自动上传到 Imgur 并获取分享链接，以便一键粘贴到论坛或聊天中

**验收标准**：
- [ ] 设置页新增"☁️ 云分享"Section → 配置上传目标（至少支持 Imgur）
- [ ] 截图工具栏增加"上传"按钮 → 上传进度条 → 完成时复制链接 + Toast 通知
- [ ] 上传目标通过 `IUploadTarget` 接口插件化，可注册多个目标
- [ ] 与动作链集成——`UploadToCloud` 节点可添加到动作链中
- [ ] 上传历史记录（链接 + 时间 + 过期时间）

## 依赖汇总（P2+P3）

| 功能 | 关键新增依赖 | 前置功能 |
|------|------------|---------|
| 日历视图 | 无 | TodoStore |
| 上下文感知 | 无 | CompactToolboxWindow |
| AI 全数据个性化 | 无 | 所有 Store 模块 + ChatManager |
| 步骤捕获 | 无 | CaptureWindow |
| 智能打码 | ZXing.Net, OpenCvSharp4（可选） | OcrService（P0） |
| 番茄钟 | 无 | TodoStore |
| 屏幕冻结 | 无 | CaptureWindow |
| QR 码识别 | ZXing.Net ≥ 0.16.9 | 无 |
| 窗口置顶 | 无 | HotkeyManager |
| 习惯打卡 | 无 | TodoStore + RepeatConfig |
| 云分享 | RestSharp/Flurl（按需） | 动作链（P1） |
