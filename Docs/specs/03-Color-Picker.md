# 屏幕取色器 产品规格说明

- **优先级**：P0
- **对应设计**：`Docs/design/features/03-Color-Picker.md`
- **版本**：v1.0

---

## 一、功能概述

为 ToolBox 增加全局屏幕取色器：用户按下热键 `Ctrl+Shift+C` 后进入取色模式，屏幕出现放大镜实时跟随鼠标，显示当前像素的精确色值（HEX/RGB/HSL 三格式）。点击鼠标左键将色值复制到剪贴板，同时记录到色值历史。该功能以极低开发成本补齐截图工具标配能力，对前端开发者、设计师和 UI 调试场景具有高频刚需价值。

| # | 用户故事 | 验收优先级 |
|---|---------|-----------|
| US1 | 作为**前端开发者**，我想要快速提取浏览器中某个元素的颜色 HEX 值，以便直接粘贴到 CSS 代码中 | P0 |

## 三、验收标准

### 取色模式

- [x] **AC1.1**：按下 `Ctrl+Shift+C` 后，全屏出现透明覆盖层，鼠标变为十字取色光标，进入取色模式
- [x] **AC1.2**：鼠标旁实时显示放大镜（120×120px），展示鼠标周围 30×30px 区域的 4 倍放大视图
- [x] **AC1.3**：放大镜中心有十字线标记，精确指示当前像素位置
- [x] **AC1.4**：放大镜旁同时显示三种格式色值：HEX（18px Bold）、RGB（11px）、HSL（11px）
- [x] **AC1.5**：取色模式下按 Tab 键切换输出格式（HEX → RGB → HSL → HEX），当前格式字号放大提示

### 取色完成

- [x] **AC2.1**：点击鼠标左键 → 当前色值（按选定格式）复制到剪贴板 → 屏幕短暂闪烁提示（圆形色块从鼠标位置缩小消失，300ms）
- [x] **AC2.2**：右下角 Toast 提示"已复制 #XXXXXX"，3 秒后自动消失
- [x] **AC2.3**：取色完成后自动退出取色模式，覆盖层消失
- [x] **AC2.4**：按 ESC 键取消取色，不复制任何内容，退出取色模式

### 色值历史

- [x] **AC3.1**：取色完成后右下角弹出小型历史面板，展示最近 10 个取色结果（两行 5 列色块，24×24 圆形）
- [x] **AC3.2**：点击历史色块 → 该色值复制到剪贴板
- [x] **AC3.3**：历史面板 3 秒无操作后自动消失
- [x] **AC3.4**：色值历史持久化到 `color_history.json`，关机重启后保留

### 多显示器与 DPI

- [x] **AC4.1**：取色器支持跨多显示器取色，放大镜正确跟随鼠标到任意屏幕
- [x] **AC4.2**：高 DPI（125%/150%/200%）屏幕下取色坐标与物理像素一致，取色值与实际显示颜色一致

## 四、功能范围

### In Scope（本期实现）

- 全局热键唤起取色模式
- 屏幕放大镜（4 倍缩放，实时跟随鼠标）
- HEX/RGB/HSL 三格式显示与切换
- 点击复制 + 闪烁提示 + Toast 通知
- 色值历史（最近 100 条，持久化）
- Tab 格式切换
- ESC 取消
- 多显示器支持
- 高 DPI 适配
- LLM 工具 `pick_color` 暴露

### Out of Scope（本期不实现）

- 截图内取色（在截图编辑器中从截图画面取色，精度更高——可作为截图标注工具栏功能后续追加）
- 色值对比/调色盘
- 取色后自动生成调色板（如提取 5 个辅色）
- 颜色名称识别（如"珊瑚红""天蓝"）
- 历史色值导出（ASE/ACO 等设计格式）

## 五、交互流程

### 主流程：取色 → 复制 → 使用

```
1. 用户按下 Ctrl+Shift+C → ColorPickerOverlay 覆盖全屏
2. 鼠标变为十字光标，放大镜跟随鼠标实时刷新（60fps）
3. 放大镜显示中心十字 + 4 倍放大画面 + HEX/RGB/HSL 三值
4. 用户移动鼠标到目标像素 → 点击左键
5. 色值（HEX #E84343）复制到剪贴板
6. 屏幕闪烁（圆形色块缩小消失）+ Toast "已复制 #E84343"
7. 色值历史面板弹出（右下角，最近 10 个色块）
8. 取色模式退出，覆盖层关闭
```

### 格式切换流程

```
1. 用户在取色模式中按 Tab
2. 输出格式从 HEX 切换到 RGB → 放大镜中当前格式字号放大（18px），其他缩小（11px）
3. 再次按 Tab → 切换到 HSL → 再按 → 回到 HEX
4. 后续点击复制时按当前选定格式输出
```

## 六、非功能需求

### 性能

- 放大镜刷新率 ≥ 30fps（鼠标移动时实时更新）
- GetPixel 单次调用 ≤ 1μs（GDI 直接读取显存）
- 取色模式热键响应 ≤ 100ms

### 安全

- 取色操作完全本地，不涉及网络通信
- 色值历史仅本地存储，不上传

### 可访问性

- 取色模式完全支持键盘操作（Tab 切换格式、ESC 退出）
- 色值显示不单独依赖颜色——同时展示文字值
- 放大镜中心十字线 2px 粗，确保弱视用户可见

### 国际化

- 色值格式名称（HEX/RGB/HSL）为国际通用缩写，无需翻译
- Toast 提示文案通过语言包切换

## 七、依赖项

| 依赖 | 类型 | 说明 |
|------|------|------|
| `gdi32.dll` GetPixel | 系统 API | 读取屏幕指定坐标像素颜色 |
| `user32.dll` GetDC/ReleaseDC | 系统 API | 获取桌面设备上下文 |
| `Graphics.CopyFromScreen` | .NET 内置 | 截取放大镜区域 |
| `HotkeyManager` | 现有模块 | 注册 `Ctrl+Shift+C` 热键 |
| `ColorPickerService` 单例 | 新建模块 | 像素读取 + 色值计算 + 格式管理 |
| `ColorHistoryStore` 单例 | 新建模块 | 取色历史持久化 |
| `ColorPickerOverlay` | 新建模块 | 全屏透明覆盖层 + 放大镜 UI |
| `ColorTools` | 新建模块 | LLM Tool 暴露 `pick_color` |

| 竞品 | 实现方式 | ToolBox 借鉴点 |
