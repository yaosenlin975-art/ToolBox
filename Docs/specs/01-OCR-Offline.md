# 离线 OCR 文字识别 产品规格说明

- **优先级**：P0
- **对应设计**：`Docs/design/features/01-OCR-Offline.md`
- **版本**：v1.0

---

## 一、功能概述

为 ToolBox 截图系统增加离线 OCR（光学字符识别）能力：用户截图后可一键提取画面中的文字，全程无需联网、无需 API Key。支持中英日韩等主流语言，识别结果可编辑、复制、翻译或发送到 AI 助手。同时在贴图窗口支持"划选区域直接识别文字"的交互。该功能将 ToolBox 截图从"视觉捕获"升级为"信息提取"，填补当前截图能力的第一空白。

## 二、用户故事

| # | 用户故事 | 验收优先级 |
|---|---------|-----------|
| US1 | 作为**开发者**，我想要截取报错弹窗后一键提取其中的错误信息文本，以便粘贴到搜索引擎或 AI 助手中快速定位问题 | P0 |
| US2 | 作为**内容编辑**，我想要截取 PDF 或不可复制的网页区域后提取文字，以便直接引用到文档中而无需手动打字 | P0 |
| US3 | 作为**多语言用户**，我想要在贴图上划选一段英文/日文区域直接获取文字，以便快速翻译或理解内容 | P1 |
| US4 | 作为**AI 助手重度用户**，我想要在对话中让 AI 调用 OCR 识别最近的截图，以便 AI 基于截图文字内容给出分析或建议 | P1 |
| US5 | 作为**历史截图管理者**，我想要右键历史截图后选择"识别文字"，以便回溯几天前截图的文字信息 | P1 |

## 三、验收标准

### 截图后 OCR

- [x] **AC1.1**：截图完成后，工具栏出现"OCR"按钮（图标为 T 带扫描线），位于标注工具栏右侧（实现为贴图右键菜单 + 热键，因项目无截图后标注工具栏）
- [x] **AC1.2**：点击 OCR 按钮后，识别过程显示加载指示器（旋转动画 + "识别中..."文字），耗时不超过 3 秒（1024×768 截图，中文混合内容）
- [x] **AC1.3**：识别完成后，截图上方叠加半透明 OCR 结果层，文本按行排列显示，点击某行文本时截图对应区域高亮（实现为独立窗口 OcrResultOverlay：左文本列表 + 右缩略图带边界框高亮，架构适配）
- [x] **AC1.4**：置信度低于 60% 的文本行以黄色/警告色标记，用户可手动修正
- [x] **AC1.5**：结果层底部提供"复制全部""翻译选中""发送到 AI""关闭"四个按钮
- [x] **AC1.6**：点击"复制全部"后，所有识别文本复制到系统剪贴板，屏幕右下角 Toast 提示"已复制 N 行文字"

### 贴图选字 OCR

- [x] **AC2.1**：在已贴图的 ScrapWindow 上，鼠标拖拽选区（蓝色半透明矩形）后松手，自动对该选区执行 OCR
- [x] **AC2.2**：OCR 结果在贴图旁弹出浮层窗口（280×200px），显示识别文字 + "复制""翻译"按钮
- [x] **AC2.3**：贴图选字模式与标注模式互斥——通过右键菜单切换 InteractionMode（View / Annotate / OcrSelect）（实现为 ScrapWindow.isOcrSelectMode bool 状态切换,不引入完整 InteractionMode 枚举,按 SOLO Agent 裁决3）

### 热键与历史

- [x] **AC3.1**：按下全局热键 `Ctrl+Shift+O` 后，对最近一次截图自动执行 OCR，结果显示在 OCR 结果覆盖层中
- [x] **AC3.2**：在 HistoryView（工作台截图页）中右键任意截图 → 菜单中出现"识别文字"选项，点击后执行 OCR

### LLM 集成

- [x] **AC4.1**：在 AI 对话中输入"识别最近截图中的文字"，LLM 能调用 `ocr_screenshot` 工具返回识别结果
- [x] **AC4.2**：`ocr_screenshot` 工具支持 `language` 参数（默认 `chi_sim+eng`），可按需指定单语言

### 语言与引擎管理

- [x] **AC5.1**：首次使用 OCR 时，若中文语言包未安装，提示用户下载（显示下载进度），下载完成后自动可用
- [x] **AC5.2**：在设置页面"OCR 文字识别"Section 中，可选择引擎（Tesseract / Windows OCR）和已安装语言（实现为引擎 RadioButton + 默认语言 TextBox + 已安装语言包只读列表展示）
- [x] **AC5.3**：Tesseract 引擎不可用时（如语言包损坏），自动回退到 Windows OCR 引擎并提示用户

## 四、功能范围

### In Scope（本期实现）

- 截图后一键 OCR（工具栏按钮 + 热键）
- 贴图划选区域 OCR
- 历史截图右键 OCR
- LLM 工具 `ocr_screenshot` 暴露
- Tesseract 主引擎 + Windows OCR 回退
- 中/英语言包管理（内置英文，中文引导下载）
- OCR 结果复制/翻译/发送到 AI
- 置信度可视化标记

### Out of Scope（本期不实现）

- PaddleOCR 高精度引擎（预留接口，v2 升级）
- 实时摄像头 OCR / 屏幕实时 OCR
- 手写文字识别专项优化
- 表格结构识别与导出
- OCR 结果翻译的离线能力（翻译依赖 LLM）
- 批量 OCR（一次识别多张历史截图）

## 五、交互流程

### 主流程：截图 → OCR → 复制

```
1. 用户按下截图热键 → CaptureWindow 出现 → 用户选区 → 截图完成
2. 截图工具栏出现，标注工具右侧显示「OCR」按钮
3. 用户点击「OCR」按钮（或按 Ctrl+Shift+O）
4. OCR 引擎异步识别（显示进度指示器）
5. 识别完成 → OcrResultOverlay 叠加在截图上
6. 用户浏览/编辑识别文本 → 点击「复制全部」
7. 文本复制到剪贴板 → Toast 提示 → OcrResultOverlay 关闭
```

### 贴图选字流程

```
1. 已有贴图 ScrapWindow → 右键菜单切换「选字模式」
2. 鼠标变为十字光标 → 用户在贴图上拖拽选区 → 松手
3. 选区区域自动 OCR → OcrTextPopup 浮层弹出
4. 用户点击「复制」→ 文本入剪贴板 → 浮层关闭
```

### LLM 调用流程

```
1. 用户截取报错弹窗
2. 用户在 AI 对话中输入："帮我看看最近截图里的错误信息"
3. LLM Agent 调用 ocr_screenshot(language="chi_sim+eng")
4. 返回 OCR 文本 → LLM 分析并给出建议
```

## 六、非功能需求

### 性能

- 1024×768 分辨率截图 OCR 耗时 ≤ 3 秒（Tesseract 引擎，中文混合内容）
- 首次加载 Tesseract 引擎 ≤ 2 秒（延迟初始化，不阻塞启动）
- 贴图区域 OCR（200×100px）耗时 ≤ 500ms
- OCR 执行期间不影响截图/贴图等主线程操作（异步后台执行）

### 安全

- 所有 OCR 处理完全本地执行，截图数据不离开用户设备
- 语言包下载通过 HTTPS，校验文件完整性（SHA256）
- OCR 结果文本不自动上传到任何远程服务

### 可访问性

- OCR 结果覆盖层支持键盘导航（Tab 切换按钮，Enter 确认）
- 置信度标记使用颜色 + 文字双重指示（不单独依赖颜色）
- 加载指示器包含文字说明，不仅依赖动画

### 国际化

- 语言包支持中/英/日/韩及组合
- UI 文案通过语言包资源字典切换（zh-CN / en-US）
- 日期/数字格式跟随系统区域设置

## 八、竞品参考

| 竞品 | 实现方式 | ToolBox 借鉴点 |
|------|---------|---------------|
| **PixPin** | 离线 OCR 毫秒级识别，贴图后可选中文本复制 | 贴图选字交互——鼠标划选即 OCR，零切换体验 |
| **ShareX** | Tesseract 集成，OCR 结果可编辑/复制 | 截图后 OCR 按钮位置（工具栏右侧），多引擎可配 |
| **Snagit** | OCR + AI 智能编辑，识别后可直接搜索文本 | 置信度低的文本标记为黄色，提示用户手动修正 |
| **CleanShot X** | OCR 后文本可搜索/复制/分享 | 结果层底部操作栏的按钮布局（复制/翻译/分享） |

结合设计文档，ToolBox 的差异化在于：(1) 贴图选字——与现有 ScrapWindow 深度整合；(2) LLM 可调用 OCR——识别结果直接入 AI 分析链路；(3) 双引擎回退——保证各种环境下的可用性。
