# 代码片段管理 产品规格说明

- **优先级**：P1
- **对应设计**：`Docs/design/features/08-Code-Snippets.md`
- **版本**：v1.0

---

## 一、功能概述

为 ToolBox 增加代码片段/文本片段管理系统。用户可保存常用的代码模板、邮件模板、命令行脚本等，每条片段关联一个触发关键字（如 `;email` 自动展开为完整邮件签名）。通过热键 `Ctrl+Shift+S` 唤起片段搜索浮窗快速粘贴，或在任意应用中输入触发关键字自动展开。工作台新增独立的片段管理页面。该功能面向开发者、运维和效率用户，与 Raycast Snippets 对标。

## 二、用户故事

| # | 用户故事 | 验收优先级 |
|---|---------|-----------|
| US1 | 作为**开发者**，我想要保存常用的代码模板（如 HTTP 请求、数据库连接字符串）并通过关键字快速插入，以便减少重复打字 | P1 |
| US2 | 作为**日常办公用户**，我想要在任意应用中输入 `;sig` 后自动展开为邮件签名，以便不用每次都复制粘贴 | P1 |
| US3 | 作为**运维人员**，我想要保存常用的命令行脚本并通过搜索快速找到并粘贴，以便在终端中执行 | P1 |
| US4 | 作为**剪贴板用户**，我想要将剪贴板历史中的内容一键保存为片段，以便将临时复制转为永久模板 | P2 |

## 三、验收标准

### 片段管理

- [ ] **AC1.1**：工作台侧边栏新增"📝 代码片段"第 7 个导航项，点击进入 SnippetView
- [ ] **AC1.2**：SnippetView 左侧显示分类列表 + 搜索框，右侧 4 列卡片网格展示片段
- [ ] **AC1.3**：每张卡片显示：片段名称（14px Bold）、触发关键字（11px Mono, AccentBrush）、内容预览（前 3 行截断）、语言标签 + 使用次数
- [ ] **AC1.4**：点击卡片 → 右侧详情面板显示完整内容（MdXaml 语法高亮渲染）+ 编辑/删除按钮
- [ ] **AC1.5**：支持"+ 新建片段"→ 弹窗编辑（名称/触发关键字/语言/分类/内容），内容区支持 Tab 键输入

### 快捷粘贴

- [ ] **AC2.1**：按下 `Ctrl+Shift+S` → 在屏幕中央弹出 SnippetPopup 搜索浮窗（360×480px, Topmost）
- [ ] **AC2.2**：浮窗顶部搜索框实时过滤片段（匹配名称/触发关键字/分类），输入响应 ≤ 100ms
- [ ] **AC2.3**：选中片段后 Enter → 展开片段内容 → 写入剪贴板 → 自动 Ctrl+V 粘贴到前台应用 → 使用次数 +1
- [ ] **AC2.4**：支持 ↑↓ 导航、Enter 粘贴、ESC 关闭

### 关键字展开

- [ ] **AC3.1**：在设置中开启关键字展开后，任意应用中输入触发关键字（如 `;sig`）+ 空格 → 自动删除关键字 → 粘贴展开的片段
- [ ] **AC3.2**：触发前缀默认为 `;`，可在设置中自定义（如 `/` 或 `@`）
- [ ] **AC3.3**：设置中可配置"排除的应用列表"（如终端、IDE），在这些应用中关键字展开不生效
- [ ] **AC3.4**：支持变量占位符 `${1:placeholder}` 语法，展开后保留占位文本供用户替换

### LLM 集成

- [ ] **AC4.1**：AI 对话中可调用 `search_snippets` 搜索片段库
- [ ] **AC4.2**：AI 对话中可调用 `get_snippet` 获取指定片段完整内容
- [ ] **AC4.3**：AI 对话中可调用 `add_snippet` 保存新片段

### 协同

- [ ] **AC5.1**：剪贴板面板（ClipboardPopup）中右键条目 → "保存为片段"→ 自动填入内容，弹出 SnippetEditor
- [ ] **AC5.2**：片段使用次数每次粘贴后自动 +1，按使用频次排序时可快速找到常用片段

## 四、功能范围

### In Scope（本期实现）

- 片段 CRUD（名称/内容/触发关键字/语言/分类）
- 触发前缀可配置（默认 `;`）
- 全局键盘钩子关键字展开
- 热键搜索浮窗快捷粘贴
- 工作台独立片段管理页面（分类 + 搜索 + 网格/列表视图）
- 语法高亮预览（复用 MdXaml）
- 变量占位符 `${1:placeholder}`
- 排除应用列表
- 使用频次统计
- 剪贴板内容一键保存为片段
- LLM 工具 `search_snippets` / `get_snippet` / `add_snippet`

### Out of Scope（本期不实现）

- 片段云同步
- 片段导入/导出
- 多用户/团队共享片段库
- 片段版本历史
- 占位符 Tab 跳转（在外部应用中无法控制光标）
- 敏感片段加密存储

## 五、交互流程

### 关键字展开流程

```
1. 用户设置中开启关键字展开，触发前缀为 ";"
2. 用户在邮件客户端输入正文："各位好，以下是今日更新。;sig "
3. SnippetExpander 键盘钩子检测到 ";sig "（触发前缀 + 关键字 + 空格）
4. SnippetStore.FindByTrigger(";sig") → 找到"邮件签名"片段
5. 自动按 Backspace 4 次（删除 ";sig "）
6. Clipboard.SetText("Best regards,\nYour Name\nTitle | ToolBox Team")
7. SendKeys.SendWait("^v") → 粘贴展开内容
```

### 快捷搜索粘贴流程

```
1. 用户按 Ctrl+Shift+S → SnippetPopup 弹出（360×480, 居中, Topmost）
2. 搜索框自动获得焦点 → 用户输入"http"
3. 列表实时过滤 → 显示"HTTP GET 请求模板""HTTP POST 请求模板"
4. 用户用 ↓ 选择"HTTP GET 请求模板"→ 按 Enter
5. 片段内容写入剪贴板 → Ctrl+V 粘贴到前台编辑器
6. SnippetPopup 关闭 → 该片段使用次数 +1
```

## 六、非功能需求

### 性能

- 关键字展开从检测到粘贴 ≤ 300ms
- 键盘钩子单次处理 ≤ 1ms（环形缓冲区 + 前缀触发）
- 500 条片段搜索过滤 ≤ 50ms

### 安全

- 键盘钩子使用 WH_KEYBOARD_LL 低层钩子（无需管理员权限）
- 仅在用户主动开启关键字展开功能时安装钩子
- 排除应用列表防止在密码管理器/终端中的误触发

### 可访问性

- SnippetPopup 完全支持键盘操作
- 片段编辑器的内容区支持 Tab 键（`AcceptsTab=True`）

## 八、竞品参考

| 竞品 | 实现方式 | ToolBox 借鉴点 |
|------|---------|---------------|
| **Raycast Snippets** | 关键字触发 + 搜索浮窗 + 变量占位符 + 使用统计 | 触发关键字自动展开 + 热键搜索浮窗双通道；变量占位符语法 |
| **Quicker** | 组合动作中的文本模板 | 片段关联触发关键字的映射方式 |
| **uTools** | 插件化文本片段管理 | 分类管理 + 卡片式展示 |

ToolBox 差异化：(1) 与 LLM 深度协同——AI 可搜索/获取/保存片段；(2) 与剪贴板联动——剪贴板内容一键转片段；(3) 双通道访问——关键字展开（无感）+ 搜索浮窗（有感）两种使用方式。
