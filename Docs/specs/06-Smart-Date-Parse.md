# 智能日期识别 产品规格说明

- **优先级**：P1
- **对应设计**：`Docs/design/features/06-Smart-Date-Parse.md`
- **版本**：v1.0

---

## 一、功能概述

为 ToolBox 待办创建流程增加智能日期识别能力：当用户在待办输入框中输入文本时，系统自动检测其中包含的自然语言日期/时间表达（如"明天下午三点开会""下周五提交报告""每周一晨跑"），实时高亮识别到的日期，并自动填入 DueDate 字段。支持正则层快速解析（毫秒级）和 LLM 辅助解析复杂表达（如"下下周二的上午"）。同时，LLM 对话中的待办创建也会自动利用日期解析能力。

## 三、验收标准

### 实时解析与高亮

- [ ] **AC1.1**：在 TodoView 快速添加输入框中输入"明天下午三点和产品经理讨论需求"，输入框内"明天下午三点"以蓝色下划线（AccentBrush, 1.5px）高亮标记
- [ ] **AC1.2**：解析时机为用户停止输入 300ms 后（防抖），解析结果不影响用户继续输入
- [ ] **AC1.3**：支持的日期表达至少包含：今天/明天/后天/大后天、周一到周日（含"下周三""这周五"）、X月X日/X.X/YYYY-MM-DD、下午N点/NN:NN、每天/每周X/每月X号

### 自动填入

- [ ] **AC2.1**：识别到日期表达后，DueDate 自动填入解析结果，TodoItem 详情面板的截止日期同步更新
- [ ] **AC2.2**：识别到重复表达（如"每周一"）后，RepeatConfig 自动填入对应的重复规则
- [ ] **AC2.3**：DateSuggestionPopup 浮层出现在输入框下方，显示"📅 截止日期: 明天 15:00"，提供快速修正按钮（[上午 09:00] [下午 15:00] [晚上 20:00]）+ [清除日期]
- [ ] **AC2.4**：用户通过修正按钮更改时间后，DueDate 即时更新
- [ ] **AC2.5**：用户点击"清除日期"后，DueDate 恢复为空，高亮消失

### 复杂表达与 LLM

- [ ] **AC3.1**：正则层无法解析的表达（如"下下周二的上午"），若用户启用了 LLM 辅助，则异步调用 LLM 解析，2-5 秒后返回结果
- [ ] **AC3.2**：LLM 辅助解析期间，输入框旁显示小型加载图标
- [ ] **AC3.3**：LLM 默认关闭（避免不必要的 API 调用），在设置页"待办"相关设置中手动开启

### LLM 工具集成

- [ ] **AC4.1**：AI 对话中调用 `add_todo` 工具时，标题参数中的日期表达自动解析并填入 DueDate
- [ ] **AC4.2**：新增 `parse_date` 工具，可独立解析文本中的日期表达

## 四、功能范围

### In Scope（本期实现）

- 正则层实时日期解析（13+ 种常见中文日期模式）
- 输入框富文本高亮（蓝色下划线）
- DueDate 自动填入 + DateSuggestionPopup 修正
- 重复规则自动提取（每天/每周/每月/每年）
- LLM 辅助解析（可选，默认关闭）
- `add_todo` 自动日期解析增强
- `parse_date` LLM 工具新增
- 设置开关：启用智能日期识别 / 启用 LLM 辅助

### Out of Scope（本期不实现）

- 语音输入日期识别
- 多语言日期解析（本期仅中文）
- 农历日期支持
- 时区转换
- 自然语言时间段解析（如"下周三到周五"）

## 五、交互流程

### 标准流程

```
1. 用户在 TodoView 输入框输入"明天下午三点讨论需求 #会议"
2. 300ms 防抖后 → SmartDateParser.Parse() 执行
3. 正则匹配到"明天下午三点"→ 解析为 DateTime.Today+1 15:00
4. 输入框中"明天下午三点"高亮为蓝色下划线
5. DateSuggestionPopup 出现在输入框下方：
   📅 截止日期: 明天 15:00
   [上午 09:00] [下午 15:00] [晚上 20:00]
   [清除日期]
6. DueDate 自动填入 2026-07-14 15:00
7. 用户按 Enter 创建 → 标题清理为"讨论需求"，DueDate 已设置，Tags 含"会议"
```

### LLM 辅助流程

```
1. 用户输入"下下周二的上午交报告"
2. 正则无法匹配 → 若开启了 LLM 辅助 → 发送到 LLM
3. LLM 返回 { "date": "2026-08-01T09:00:00", "confidence": 0.85 }
4. DateSuggestionPopup 显示解析结果 + 置信度标记
5. 用户确认 → DueDate 填入
```

## 六、非功能需求

### 性能

- 正则层解析延迟 ≤ 1ms（启动时预热编译正则）
- 防抖 300ms 确保不频繁触发解析
- LLM 辅助解析超时设为 10 秒

### 安全

- LLM 辅助解析时，仅发送需要解析的日期文本片段（不含完整待办内容）
- 正则解析完全本地执行

### 可访问性

- 高亮使用颜色 + 下划线双重标识
- DateSuggestionPopup 支持键盘操作（Tab 切换修正按钮，Enter 确认）

### 国际化

- 本期仅支持中文日期表达
- 英文日期解析作为 v2 扩展（预留接口）
