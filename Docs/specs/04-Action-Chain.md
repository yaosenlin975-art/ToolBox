# 截图后自动化动作链 产品规格说明

- **优先级**：P1
- **对应设计**：`Docs/design/features/04-Action-Chain.md`
- **版本**：v1.0

---

## 一、功能概述

为截图完成后的处理增加可配置的"动作链"——用户可定义截图后自动执行的一系列操作，例如"OCR 提取文字 → 翻译为中文 → 复制结果"。动作链由多个步骤节点串联而成，像一个可定制的截图后处理流水线。内置 3 条开箱即用的默认链，用户可在设置中可视化管理、创建和编辑动作链。执行进度在迷你窗口底部显示，随时可取消。

## 二、用户故事

| # | 用户故事 | 验收优先级 |
|---|---------|-----------|
| US1 | 作为**国际化团队成员**，我想要截取英文界面后自动 OCR + 翻译为中文 + 复制结果，以便快速理解外语界面内容 | P1 |
| US2 | 作为**文档编写者**，我想要截图后自动保存到指定文件夹 + 复制到剪贴板，以便同时保留文件归档和粘贴使用 | P1 |
| US3 | 作为**效率控**，我想要一键截图后自动"发送到 AI 助手分析"，以便省去手动切换窗口粘贴截图的步骤 | P1 |
| US4 | 作为**高级用户**，我想要自定义截图后的处理步骤组合，以便适配不同的工作场景（如有的场景只复制，有的场景需要 OCR） | P2 |

## 三、验收标准

### 默认动作链

- [ ] **AC1.1**：内置 3 条默认链：(a)"仅复制到剪贴板"(b)"OCR → 复制文字"(c)"保存到文件 + 复制到剪贴板"
- [ ] **AC1.2**：用户第一次使用时，默认选中"仅复制到剪贴板"作为当前默认链（与现有截图行为一致，不改变用户习惯）
- [ ] **AC1.3**：截图完成工具栏底部显示当前默认动作链名称（如 `⚡ OCR→翻译→复制`），点击可展开下拉菜单切换链

### 链执行

- [ ] **AC2.1**：截图完成后自动按默认链的节点顺序执行，每个节点执行时迷你窗口底部显示节点名称进度（如"OCR 识别中..."）
- [ ] **AC2.2**：链执行中任意节点失败时，默认停止后续节点并提示错误（可配置为"忽略错误继续"）
- [ ] **AC2.3**：迷你窗口底部显示执行进度条（AccentBrush，4px 高），用户可点击进度条旁的 × 按钮中途取消
- [ ] **AC2.4**：链执行完成后，迷你窗口 Toast 通知执行结果（如"动作链「OCR→复制」完成——已复制 3 行文字"）

### 动作链编辑器

- [ ] **AC3.1**：设置页"🔗 截图动作链"Section 中，卡片式列表展示所有链（名称 + 节点图标串联如 📸→🔤→📋）
- [ ] **AC3.2**：点击链进入详情编辑：左侧可用节点列表（9 种节点类型）、中间链画布（节点卡片从上到下排列 + 箭头连线）、右侧选中节点的参数配置
- [ ] **AC3.3**：支持从可用节点列表拖拽节点到链画布中，或通过下拉选择添加节点
- [ ] **AC3.4**：支持"+ 新建链""设为默认""重置内置链"操作
- [ ] **AC3.5**：链长度限制最大 20 个节点（防止循环依赖）

### LLM 集成

- [ ] **AC4.1**：AI 对话中可使用 `list_action_chains` 工具列出所有可用链
- [ ] **AC4.2**：AI 对话中可使用 `create_action_chain` 工具创建新链（指定名称 + 节点类型列表）

## 四、功能范围

### In Scope（本期实现）

- 3 条内置默认动作链
- 截图后自动执行当前默认链
- 9 种节点类型：保存文件、复制到剪贴板、OCR 提取、翻译文本、发送到 AI、打开编辑器、添加水印、调整尺寸、条件分支
- 线性链 + 条件分支链
- 设置页可视化链编辑器（列表 + 详情编辑）
- 执行进度显示 + 中途取消
- 遇错即停 / 忽略错误继续（可配置）
- LLM 工具 `list_action_chains` / `create_action_chain`

### Out of Scope（本期不实现）

- 并行链（多个节点同时执行）
- 节点拖拽画布（当前为列表式编辑，非流程图式自由拖拽）
- 社区共享/导入导出动作链
- 上下文感知自动推荐动作链
- 云上传节点（依赖 P3 云分享功能）
- 链执行历史与回放

## 五、交互流程

### 默认使用流程

```
1. 用户截图完成 → CaptureWindow.OnCaptureComplete(bitmap)
2. 读取 ActionChainStore 当前默认链（如"OCR→翻译→复制"）
3. ActionChainEngine 依次执行节点：
   a. OcrExtractNode → OcrService.RecognizeAsync() → 得到文字
   b. TranslateTextNode → ChatManager 翻译 → 得到中文
   c. CopyToClipboardNode → Clipboard.SetText(中文)
4. 迷你窗口 Toast："动作链「OCR→翻译→复制」完成"
```

### 自定义链流程

```
1. 用户打开设置 → 截图动作链 Section
2. 点击 "+ 新建链" → 输入名称"我的备用链"
3. 从可用节点列表添加节点：OCR 提取 → 保存文件
4. 配置保存文件节点的参数：目标文件夹、文件命名规则
5. 点击保存 → 新链出现在列表中
6. 在截图工具栏下拉菜单中可切换到"我的备用链"
```

## 六、非功能需求

### 性能

- 链执行引擎启动延迟 ≤ 50ms（节点实例化）
- 单个节点切换到下一个节点延迟 ≤ 30ms
- 执行进度回调间隔 ≥ 100ms（避免 UI 刷新过于频繁）

### 安全

- 文件保存节点限制在用户配置的目标文件夹内（不超出白名单范围）
- 翻译/SendToAi 节点涉及 LLM 调用，提示用户"此操作将发送截图/文本到 AI 供应商"

### 可访问性

- 链编辑器支持键盘导航（Tab 切换节点和参数）
- 执行状态文本 + 进度条双重反馈

### 国际化

- 内置链名称和节点名称通过语言包切换
- 日期/时间参数格式跟随系统区域设置

## 八、竞品参考

| 竞品 | 实现方式 | ToolBox 借鉴点 |
|------|---------|---------------|
| **ShareX** | After-Capture/After-Upload 工作流，可配置截图后自动执行多个操作，80+ 上传目标 | 链的概念——截图后自动执行预设操作序列；内置默认链降低使用门槛 |
| **Greenshot** | 多目标分发——截图后同时复制到剪贴板/保存文件/打开编辑器 | "保存+复制"并行输出的常用模式 |
| **Quicker** | 组合动作编辑器——积木式拖拽编排 | 可用节点列表 + 参数配置的编辑模式 |

ToolBox 差异化：(1) 与 OCR 模块深度联动——链节点可直接调用 OCR 结果；(2) LLM 参与的智能链——翻译/SendToAi 节点利用 AI 能力，而非简单文本处理；(3) 迷你窗口进度反馈——执行进度实时可见。
