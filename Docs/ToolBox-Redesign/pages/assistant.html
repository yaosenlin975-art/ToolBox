我在做一个 WPF 的 MVVM 项目，发现 ListBox 的 ItemsSource 绑定了 ObservableCollection，但 UI 没有更新。帮我看看可能是什么原因？
14:23

常见的原因有以下几种，我帮你逐一排查：

1. 集合引用被替换 — 如果你在 ViewModel 中对整个集合重新赋值（= new ObservableCollection），而不是调用 Add/Remove，并且属性没有触发 PropertyChanged，UI 就不会刷新。

2. 绑定路径错误 — 确认 XAML 中的 ItemsSource="{Binding Items}" 路径是否正确匹配属性名。

3. DataContext 未设置 — 确保 Window.DataContext 或 UserControl.DataContext 指向了正确的 ViewModel 实例。

找到问题了！在 MainViewModel.cs 第 18 行，你的 setter 中没有调用 OnPropertyChanged。修复如下：

public ObservableCollection<TaskItem> Tasks { get => _tasks; set { _tasks = value; OnPropertyChanged(nameof(Tasks)); // <-- 添加这一行 } }

不过更推荐的做法是不要替换整个集合，而是在初始化时创建一次，后续只用 Tasks.Add() / Tasks.Remove() 操作，这样 ObservableCollection 自身的 CollectionChanged 事件会自动通知 UI。
